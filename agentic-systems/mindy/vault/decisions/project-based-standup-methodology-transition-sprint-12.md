# Project-Based Standup Methodology Transition (Sprint 12)

*Decision made: 2026-04-21*

## Context
Team was using user-based standup format which created challenges in understanding project progress, task ownership, and cross-functional coordination as projects increased in complexity and team size grew.

## Decision
Transition from user-based to project-based standup methodology:
- Organize daily standups around active projects rather than individual contributors
- Use project board filters to improve visibility on completed tasks and project context
- Implement weekly backlog refinements to prevent task stagnation and maintain momentum
- Encourage breakout sessions for deeper technical discussions outside standup time
- Maintain clear task ownership while organizing updates by project impact
- Establish shared project goals and milestones for better coordination

## Rationale
- Improves visibility into project progress and cross-functional dependencies
- Enhances understanding of how individual contributions impact overall project success
- Reduces confusion about task ownership and project status
- Supports better coordination across multiple concurrent projects
- Enables more effective identification of blockers and resource allocation needs
- Aligns with engineering tech lead role establishment for structured project management

## Implementation
- Luis Suarez leading transition as Engineering Tech Lead with standup facilitation responsibilities
- Project board filters configured to show project-driven task organization
- Weekly backlog refinement sessions established for proactive task management
- Team training on project-focused update format and coordination processes
- Regular evaluation and refinement of standup methodology effectiveness

## Status
Active implementation
